What is Kubernetes?
  • Also known as 'k8s'
  • Kubernetes is an extensible, portable, and open-source platform designed by Google in 2014
  • Kubernetes is a container management and orchestration system
  • It runs and manages containerized applications on a cluster
  • Provides container grouping, load balancing, auto-healing, scaling features

    • Using Kubernetes we can do cool stuff like :
    • Manage containerized applications across a cluster of nodes
    • Application auto scaling
    • Efficient & Robust deployment of application
    • Self healing application
    • Blue/green deployment, canary deployment-Easy Service deployments, updates and scalability
    • Long running services, but also batch (one-off) jobs
    • Overcommit our cluster and evict low-priority jobs
    • Run services with state full data (databases etc.)
    • Fine-grained access control defining what can be done by whom on which resources
    • Integrating third party services (service catalog)
    • Automating complex tasks (operators)

    Key Features
    Container Grouping - Pods
    • A Pod is a group of containers that share the storage, users, network interfaces, etc. using Linux namespaces (ipc, uts, mount, pid, network and user), cgroups, and other kernel features.
    • Containers in a pod share an IP address and the port space

    Container Orchestration - Replication Controller
    • Replication controller is a logical entity that creates and manages pods.
    • Replication controllers auto heal pods according to the given health checks.
    • These health checks are called liveness probes.
    • Replication controllers support manual scaling of pods, and this is handled by the replica count.


    Health Checking
    Software applications fail due to many reasons:
    • bugs in the code
    • resource limitations
    • networking issues
    • infrastructure problems, etc.

    Monitoring software application deployments is essential.
    Kubernetes (via Kubelet agent) provides two main mechanisms for monitoring applications.

    1. Process Health Checking
    • Kubelet continuously checks the health of the containers via the Docker daemon
    • If a container process is not responding, it will get restarted.
    • This feature is enabled by default and it’s not customizable

    2. Application Health Checking
    Kubernetes provides three methods for monitoring the application health and these are known as health checking probes.

    Service Discovery and Load Balancing

    • Kubernetes service provides a mechanism for load balancing pods.
    • It is implemented using kube-proxy
    • Internally uses iptable rules for load balancing at the network layer.
    • Each Kubernetes service exposes a DNS entry for accessing the services within the Kubernetes internal network

    Automated Rollouts and Rollbacks

    Horizontal Autoscaling
    • Horizontal Pod Autoscalers provide autoscaling capabilities for pods.
    • It does this by monitoring health statistics sent by the cAdvisor.
    • A cAdvisor instance runs in each node and provides information on CPU, memory, and disk usage of containers.
    • These statistics get aggregated by Heapster and get accessible via the Kubernetes API server.
    • Currently, horizontal autoscaling is only available based on CPU usage.
    • Work is in progress to support custom metrics.
